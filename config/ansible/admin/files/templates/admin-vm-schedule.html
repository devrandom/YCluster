VM Schedule

Declare when your VMs should be up; each host converges within ~1 minute. Windows are one-shot (start – end, multiple allowed, times shown in your local timezone); set the end directly or via the hours field. A ⏳ pending tag shows when a VM's live state differs from what the schedule wants now. Scheduled stops warn the guest, wait a grace period, then release the VM's GPUs back to the host pool (re-attached at the next start) — Stop now skips the remaining grace. Saves are checked against host GPU capacity, shown in the commitments below. Scheduler-started runtime is billed to the owner; unmanaged means the scheduler never touches the VM (its GPUs stay held); switching modes keeps your windows for later.
